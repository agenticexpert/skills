## Formats

### `vision.md`

```markdown
# Vision

## Why
[The problem being solved. One or two sentences. Honest and specific.]

## What
[What is being built. Not features ‚Äî the outcome.]

## Who
[Who it's for. Be specific.]

## Success Looks Like
[Concrete, observable outcomes that signal the vision is achieved.]
```

### `pillars.md`


## Commands

### `thinky init`

Guided interview to originate the vision:

1. Check if `agents/docs/thinky/` already exists ‚Äî if so, warn and ask before overwriting
2. Ask the user:
   - Why does this project exist? What problem does it solve?
   - What is being built? (outcome, not features)
   - Who is it for?
   - What does success look like?
   - What are the non-negotiables? (pillars)
   - What are the top 1‚Äì3 goals?
3. Write `vision.md`, `pillars.md`, `goals.md` from answers
4. Create empty `parked.md` and `coverage.md`
5. Confirm and show `thinky show`

### `thinky show`

Display the full vision in this order:
1. Why (from `vision.md`)
2. What + Who (from `vision.md`)
3. Pillars (from `pillars.md`)
4. Goals with coverage status (from `goals.md` + `coverage.md`)

Keep it scannable ‚Äî use the file content directly, don't summarize.

### `thinky why`

Read and display only the `## Why` section from `vision.md`.

### `thinky add goal "<text>"`

1. Read `goals.md`
2. Assign next ID (G01, G02, ...)
3. Append new row with status `uncovered`
4. Confirm

### `thinky add pillar "<text>"`

1. Read `pillars.md`
2. Append new pillar with next number
3. Ask: why is this non-negotiable? (one line)
4. Write with reason
5. Confirm

### `thinky align`

1. Read `goals.md`
2. Read `agents/docs/tasky/roadmaps/index.md` for all roadmaps
3. For each roadmap, ask (or infer from roadmap name + milestones) which goals it serves
4. Write `coverage.md`
5. Display coverage table

### `thinky gaps`

1. Run align (or read existing `coverage.md`)
2. Display only goals with status `uncovered`
3. Suggest: "Consider a roadmap for: [goal]"

# thinky — Vision & Intent Origination

Originate and maintain the vision that coordinates all project work. Captures the *why* and *what* before anything is built. Feeds tasky roadmaps with purpose and keeps them coherent.

## Usage

**Setup**
```
/thinky init
```

**Vision**
```
/thinky show
/thinky why
/thinky add goal "<text>"
/thinky add pillar "<text>"
```

**Coordination**
```
/thinky align
/thinky gaps
```

**Parking Lot**
```
/thinky park "<idea>"
/thinky triage
```

- Keep answers short and scannable
- Show the vision as the north star — everything else serves it

## Project Configuration: THINKY.md

Thinky supports a project-level configuration file at the **git root**: `THINKY.md`.

### What it can contain

- **Data directory**: if not using the default `agents/docs/thinky/`
- **Default roadmap tool**: e.g. always coordinate with tasky
- **Conventions**: project-specific vision conventions or standing rules

### `thinky init`

Creates both `THINKY.md` and the vision structure interactively:

1. Check whether `THINKY.md` already exists — if so, warn and ask before overwriting
2. Ask the user:
   - Project / product name
   - Data directory (default: `agents/docs/thinky/`)
   - Any standing conventions for this project's vision work
3. Write `THINKY.md` from template
4. Continue into vision origination (why, what, who, pillars, goals)

Template written by `init`:

```markdown
# THINKY — Project Configuration

> This file is read by thinky at the start of every session.

## Project

- **Name**: {project name}
- **Data directory**: {data directory}

## Conventions

{user-supplied conventions, or placeholder text}
```

## Startup

**Before executing any thinky command**, run this startup sequence:

1. Determine the project root (directory containing `.git/`, or working directory)
2. Check whether `{project_root}/THINKY.md` exists
3. If it exists and has **not** been read yet in this session:
   - Read it
   - Treat its contents as authoritative project-level instructions
   - These instructions **extend and override** skill defaults where they conflict
4. Proceed with the requested command

If `THINKY.md` is not found, continue with skill defaults — no warning needed.

## How It Works

Detects intent and routes to the appropriate reference:
- Vision commands → `references/vision.md`
- Parking lot commands → `references/parked.md`

## Commands

### Setup

| Command | Description |
|---------|-------------|
| `init` | Guided interview to originate the vision — creates `agents/docs/thinky/` structure |

### Vision

| Command | Description |
|---------|-------------|
| `show` | Display vision, pillars, and goals |
| `why` | Just the why statement |
| `add goal "<text>"` | Add a measurable outcome to goals |
| `add pillar "<text>"` | Add a non-negotiable constraint |

### Coordination

| Command | Description |
|---------|-------------|
| `align` | Map existing tasky roadmaps to goals — shows which goals each roadmap serves |
| `gaps` | Goals with no roadmap coverage — where work is missing |

### Parking Lot

| Command | Description |
|---------|-------------|
| `park "<idea>"` | Fast-capture a future roadmap idea — no commitment, no structure required |
| `triage` | Review parked items — promote to tasky roadmap or discard |

## Files

```
THINKY.md                       # Project config (git root) — created by init

.claude/skills/thinky/
├── SKILL.md                    # This file
└── references/
    ├── vision.md               # Vision commands and formats
    └── parked.md               # Parking lot commands and formats

agents/docs/thinky/
├── vision.md                   # Why, what, who — the north star
├── pillars.md                  # Non-negotiable constraints
├── goals.md                    # Measurable outcomes — what done looks like
├── parked.md                   # Ideas not yet roadmapped
└── coverage.md                 # Goal → roadmap mapping (generated by align)
```

## Intent Detection

1. Parse command prefix or natural language intent:
   - `init`, `show`, `why`, `add` → load `references/vision.md`
   - `align`, `gaps` → load `references/vision.md`
   - `park`, `triage` → load `references/parked.md`
   - Natural language like "what's the vision", "why are we building this", "what are our goals" → load `references/vision.md`
   - Natural language like "I just thought of something", "parking an idea", "don't forget" → load `references/parked.md`
2. Execute with the reference as context
